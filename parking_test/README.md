# YOLOv8 Parking Occupancy Fine-Tuning

This project is now focused only on the custom summer parking dataset from `my_dataset_summer.zip`.
The old PKLot/Roboflow/lightweight dataset code and generated outputs were removed.

## Dataset Format

The zip is expected to contain:

```text
my_dataset_summer/
├── ParkingDataset-1/
│   └── ... images ...
└── Labels_Output/
    └── ... matching YOLO .txt labels ...
```

Images and labels must share the same relative path and filename stem. Example:

```text
ParkingDataset-1/2026-03-31/05/21-08_heartbeat.jpg
Labels_Output/2026-03-31/05/21-08_heartbeat.txt
```

Classes:

```text
0: empty
1: occupied
```

## Project Structure

```text
parking-occupancy-detection/
├── dataset/                    # generated locally, ignored by git
├── models/
│   └── pretrained_public_best.pt
├── runs/                       # generated training/evaluation outputs
├── src/
│   ├── prepare_custom_dataset.py
│   ├── train.py
│   ├── evaluate.py
│   ├── predict.py
│   ├── compare_runs.py
│   └── utils.py

├── README.md
```

`models/pretrained_public_best.pt` is the starting checkpoint for fine-tuning. It is the only kept artifact from the earlier work because the new training uses it as pretrained weights.

## Install On Windows

```bat
cd parking-occupancy-detection
python -m venv env
env\Scripts\activate
pip install -r requirements.txt
```

For CUDA, install the PyTorch build that matches your NVIDIA driver from the official PyTorch instructions, then verify:

```bat
python -c "import torch; print(torch.cuda.is_available())"
```

## Prepare The Dataset

```bat
python src\prepare_custom_dataset.py --zip C:\Users\YOUR_USER\Downloads\my_dataset_summer.zip
```

This creates:

```text
dataset/summer_yolo/train/images
dataset/summer_yolo/train/labels
dataset/summer_yolo/val/images
dataset/summer_yolo/val/labels
dataset/summer_yolo/test/images
dataset/summer_yolo/test/labels
data_summer.yaml
```

## Train / Fine-Tune

Recommended CUDA run:

```bat
python src\train.py ^
  --prepare ^
  --zip C:\Users\YOUR_USER\Downloads\my_dataset_summer.zip ^
  --weights models\pretrained_public_best.pt ^
  --device 0 ^
  --imgsz 640 ^
  --epochs 80 ^
  --batch 8 ^
  --patience 15
```

CPU fallback:

```bat
python src\train.py --prepare --device cpu --epochs 20 --batch 4
```

Training outputs:

```text
runs/summer_finetune/results.csv
runs/summer_finetune/results.png
runs/summer_finetune/confusion_matrix.png
runs/summer_finetune/weights/best.pt
models/fine_tuned_summer_best.pt
models/fine_tuned_summer_last.pt
```

The training script prints:

- Precision
- Recall
- mAP@0.5
- mAP@0.5:0.95
- Accuracy note for object detection
- Confusion matrix output path

## Evaluate

```bat
python src\evaluate.py ^
  --model models\fine_tuned_summer_best.pt ^
  --data data_summer.yaml ^
  --split test ^
  --device 0
```

## Test On A Separate Zip Dataset

Use this when you have a new labeled zip dataset and want to test the saved best model without training again:

```bat
python src\test_on_zip_dataset.py ^
  --zip "C:\Users\YOUR_USER\Downloads\untitled folder.zip" ^
  --model models\fine_tuned_summer_best.pt ^
  --device 0
```

CPU fallback:

```bat
python src\test_on_zip_dataset.py --zip "C:\Users\YOUR_USER\Downloads\untitled folder.zip" --device cpu
```

This script:

- extracts the zip into `runs/new_dataset_test/prepared`
- finds the image folder and `Labels_Output` folder
- creates a temporary YOLO test dataset
- prints official YOLO metrics: Precision, Recall, mAP@0.5, mAP@0.5:0.95
- prints the comparison-style accuracy report based on ground-truth parking boxes
- saves confusion matrices and plots under `runs/new_dataset_test`

## Predict

```bat
python src\predict.py ^
  --model models\fine_tuned_summer_best.pt ^
  --source dataset\summer_yolo\test\images ^
  --output runs\summer_predict ^
  --device 0
```

Predicted images and YOLO txt outputs are saved under `runs/summer_predict`.

## Compare Two Runs

If you train multiple versions, compare their `results.csv` files:

```bat
python src\compare_runs.py ^
  --first runs\summer_finetune_old\results.csv ^
  --second runs\summer_finetune\results.csv ^
  --output-dir runs\comparison
```

Outputs:

```text
runs/comparison/metrics_comparison.csv
runs/comparison/metrics_comparison.png
runs/comparison/loss_curves_comparison.png
```

## Notes

- `dataset/` and `runs/` are generated folders.
- `data_summer.yaml` is generated by `prepare_custom_dataset.py`.
- YOLO object detection does not have a single standard accuracy metric like image classification; precision, recall, and mAP are the primary quality metrics.
